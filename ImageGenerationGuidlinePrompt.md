# 🧠 INSIGHTSTREAM VISUAL SYSTEM (v1.0)

Aligned with your blog engine  and your positioning:

> AI = System layer
> Conversations → Signals → Insights → Growth

---

# 🎯 1. DESIGN PRINCIPLES (NON-NEGOTIABLE)

## 1. Clarity > Beauty

* Diagram must be understandable in **3–5 seconds**
* If it looks cool but unclear → reject

---

## 2. System > Illustration

* Always show:

  * Flow
  * Components
  * Relationships

---

## 3. Consistency > Creativity

* Same structure across blogs
* Build **recognition over time**

---

## 4. Dense but Structured

* You *want* information-rich visuals
* But:

  * Group logically
  * Avoid clutter

---

# 🧩 2. CORE VISUAL LANGUAGE (STANDARD BLOCKS)

Every diagram uses these primitives:

---

## 🔹 Entities (Boxes)

| Type          | Meaning                       |
| ------------- | ----------------------------- |
| Square box    | System / Tool                 |
| Rounded box   | Layer (Signal, Context, etc.) |
| Circle        | Loop step                     |
| Big container | Unified system                |

---

## 🔹 Arrows

| Type       | Meaning                  |
| ---------- | ------------------------ |
| Straight → | Flow                     |
| Curved ↺   | Loop                     |
| Dotted →   | Weak / broken connection |

---

## 🔹 Layers (Standard Names)

Always reuse:

```
Signal
Context
Decision
Action
Learning
```

👉 Never rename these → builds recognition

# 🏗️ 3. YOUR 3 CORE IMAGE TYPES

These will cover 90% of your blogs.

---

# 🥇 TYPE 1 — “BEFORE → AFTER SYSTEM”

### Use for:

* Campaign vs system
* Tools vs systems
* Manual vs AI

---

## Structure:

```
[LEFT: Problem] → [RIGHT: System]
```

---

## Layout:

### LEFT (Fragmented world)

* Ads
* CRM
* Chatbot
* Email
* Human bottleneck

### RIGHT (Unified system)

```
Signals → Decision Layer → Actions
           ↓
        Feedback Loop
```

---

## Prompt Template

```text
Hand-drawn whiteboard style infographic on a clean off-white background.

LEFT SIDE:
Fragmented SMB tools:
- Ads (campaign)
- Chatbot (reactive)
- CRM (stores data)
- Email (manual follow-up)
Show broken/dotted connections.
Include a stressed founder (human bottleneck).

RIGHT SIDE:
Unified AI system:
- Inputs: Website clicks, chats, emails
- Central box: "Decision Layer (AI)"
- Outputs: Respond, qualify, book demo, update CRM

BOTTOM:
Feedback loop clearly shown:
Signal → Context → Decision → Action → Learning → Repeat

Style:
Minimal, sketch-like, clean, structured, no shiny effects.
```

# 🎨 4. STYLE GUIDE (VERY IMPORTANT)

## Color Palette

* Background: off-white / cream
* Primary: dark gray / black
* Accent:

  * Blue (system)
  * Green (learning)
  * Orange (decision)

---

## Typography

* Handwritten / sketch font
* Clean sans-serif fallback

---

## Visual Density Rule

| Element       | Limit |
| ------------- | ----- |
| Boxes         | 6–10  |
| Arrows        | 6–12  |
| Words per box | ≤ 5   |

---

# ⚠️ 5. COMMON MISTAKES (AVOID)

## ❌ Wrong loop order

Always:

```
Signal → Context → Decision → Action → Learning → Repeat
```

---

## ❌ Too shiny / futuristic

* No neon
* No glowing circuits
* No sci-fi

---

## ❌ Too abstract

* Always include:

  * Real examples (chat, email, CRM)

---

## ❌ Too text-heavy

* Diagram ≠ blog
* Keep labels tight
